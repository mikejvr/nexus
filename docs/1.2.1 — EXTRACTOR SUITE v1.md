# **PHASE‑1 ENGINE 1 — EXTRACTOR SUITE v1 (Constitution‑Aligned Revision)**  
### *“The minimal, structural, deterministic truth‑extraction layer.”*

Extractor Suite v1 is the **first engine** in the Phase‑1 deterministic substrate.  
Its job is to extract **only** the minimal, structural, deterministic facts from a file — nothing more, nothing less.

This engine defines the **ground truth** for every asset in NEXUS‑I.

---

# **1. RESPONSIBILITIES**

Extractor Suite v1 must:

### **✔ Extract minimal, structural, deterministic fields**
Only fields that can be derived **directly and deterministically** from the file’s raw bytes.

### **✔ No timestamps from filesystem**
Forbidden:
- creation time  
- modification time  
- access time  
- inode timestamps  

### **✔ No nondeterministic metadata**
Forbidden:
- OS metadata  
- environment variables  
- locale‑dependent values  
- random values  
- UUIDs  

### **✔ No inference**
Extractors must not guess or infer meaning.

### **✔ No heuristics**
No probabilistic or semantic classification.

### **✔ No environment‑dependent behavior**
Output must be identical across:
- machines  
- OSes  
- locales  
- Python versions  

### **✔ No semantic analysis**
Extractors do **not** interpret content — only structure.

---

# **2. OUTPUT SCHEMA (Constitutional Form)**

Every extractor must output **exactly** this structure:

```json
{
  "asset_id": "<content_hash>",
  "source_path": "<relative_path>",
  "file_type": "<canonical_type>",
  "size_bytes": <int>,
  "hash_sha256": "<sha256>",
  "extractor": "<name>",
  "extractor_version": "<semver>"
}
```

### **Field Definitions**

| Field | Meaning |
|-------|---------|
| **asset_id** | SHA‑256 of file content (content hash). |
| **source_path** | Relative path inside intake directory. |
| **file_type** | Canonical MIME or NEXUS type. |
| **size_bytes** | Exact byte length of file. |
| **hash_sha256** | Duplicate of asset_id (explicit content hash). |
| **extractor** | Name of extractor (e.g., `"pdf"`, `"image"`). |
| **extractor_version** | Semantic version of extractor implementation. |

### **Schema Stability**
This schema is **frozen** for Phase‑1.  
Any change requires a constitutional amendment.

---

# **3. REMOVED FIELDS (Explicitly Forbidden)**

Extractors must **never** include:

### ❌ `created`  
### ❌ `modified`  
### ❌ any filesystem timestamp  
### ❌ any nondeterministic metadata  
### ❌ OS‑specific metadata  
### ❌ environment‑dependent values  
### ❌ semantic fields (title, author, EXIF, etc.)  
### ❌ warnings or errors that depend on runtime environment  

These fields violate determinism and are constitutionally prohibited.

---

# **4. CONTROLLED VOCABULARY FOR `file_type`**

The `file_type` field must be one of:

### **Option A — MIME Type**
Examples:
- `application/pdf`
- `image/png`
- `text/plain`
- `application/zip`

### **Option B — NEXUS Canonical Type**
Defined in **Phase‑1 Appendix A**.

Examples:
- `nexus/binary`
- `nexus/unknown`
- `nexus/text-utf8`
- `nexus/image-structural`

### **Rules**
- Must be lowercase.  
- Must be deterministic.  
- Must not depend on OS MIME databases.  
- Must not use heuristics.  
- Must not use content inference beyond structural signatures.

---

# **5. EXTRACTOR PROPERTIES**

Extractors must be:

### **✔ Pure**
No side effects, no global state.

### **✔ Stateless**
No caching, no memoization, no cross‑file influence.

### **✔ Deterministic**
Same file → same output, byte‑for‑byte.

### **✔ Schema‑Stable**
Output must always match the extractor schema.

### **✔ Minimal**
Only structural fields allowed.

### **✔ Non‑semantic**
No interpretation of content.

### **✔ Stdlib‑Only**
No external libraries in Phase‑1 extractors.

---

# **6. WHAT EXTRACTORS ARE ALLOWED TO DO**

Extractors may:

- read file bytes  
- compute SHA‑256  
- compute file size  
- detect file type using deterministic rules  
- parse minimal structural metadata (e.g., PNG width/height)  
- validate structural integrity (e.g., ZIP header)  

These operations are deterministic and structural.

---

# **7. WHAT EXTRACTORS ARE *NOT* ALLOWED TO DO**

Extractors may **not**:

- read filesystem timestamps  
- read OS metadata  
- read environment variables  
- use randomness  
- use non‑stdlib libraries  
- perform semantic analysis  
- infer content meaning  
- decode complex formats  
- include nondeterministic warnings  
- include nondeterministic error messages  

Any violation is a constitutional breach.

---
